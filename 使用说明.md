# AI Session Explorer 使用说明

这份说明写给第一次使用的人。你只需要理解一件事：**工具和数据分开保存**。

## 这个工具解决什么问题

你在不同电脑上使用 Codex / Claude Code 时，会产生很多本地聊天记录。AI Session Explorer 用来：

- 同步这些聊天记录
- 搜索历史对话
- 查看当前工作目录、最近对话、完整详情
- 用 Kimi 总结长对话
- 必要时跳到 claude-replay editor 回放

## 为什么要分成两个仓库

| 仓库 | 是否公开 | 内容 |
| --- | --- | --- |
| 工具仓库 `ai-session-explorer` | 可以公开 | 页面、脚本、服务、说明 |
| 数据仓库 `ai-session-data` | 必须私有 | 你的真实聊天记录和总结 |

这样以后你可以开源工具，但不会公开自己的聊天记录。

当前新仓库只支持这种新模式，不再兼容旧的“工具仓库里直接放 data”的模式。

## 第一次配置

### 第一步：把两个仓库放在一起

推荐放在同一个目录下，例如：

```text
D:\00容器\ai_sys\
  ai-session-explorer\
  ai-session-data\
```

其中：

- `ai-session-explorer` 是公开工具仓库
- `ai-session-data` 是私有数据仓库

### 第二步：准备 `.env.local`

进入工具仓库 `ai-session-explorer`，复制配置模板：

```powershell
copy .env.local.example .env.local
```

打开 `.env.local`，配置：

```env
SESSION_DATA_ROOT=D:\00容器\ai_sys\ai-session-data\data
```

这项是必须的。它告诉工具：真实数据应该放到私有数据仓库。

### 第三步：可选配置 Kimi

如果要用“会话总结”，继续在 `.env.local` 里配置：

```env
SUMMARY_BASE_URL=https://你的-kimi-compatible-api
SUMMARY_API_KEY=你的-key
SUMMARY_MODEL=kimi-for-coding
SUMMARY_API_FORMAT=openai
SUMMARY_INPUT_MAX_CHARS=120000
```

如果不配置 Kimi，搜索和查看历史仍然可以正常用。

## 日常只点三个文件

### 1. 到公司或回家后，先点 Start

双击：

```text
Start-AISessionWork.bat
```

效果：

- 拉取最新工具代码
- 拉取最新私有数据
- 把私有数据导入到当前电脑的 Codex / Claude Code

也就是说，换电脑后先点它，当前电脑就能拿到另一台电脑同步过来的历史。

### 2. 要查历史时，点 Open

双击：

```text
Open-SessionExplorer.bat
```

效果：

- 启动本地服务
- 打开 `http://127.0.0.1:8788/`
- 在网页里搜索、筛选、查看详情、生成总结

新仓库默认使用 `8788`，旧项目可以继续使用 `8787`，这样你可以同时打开两个页面对比测试。

网页里的“刷新最新记录”会把当前电脑刚产生的新对话导出到私有数据目录，并刷新列表。

### 3. 结束工作前，点 Finish

双击：

```text
Finish-AISessionWork.bat
```

效果：

- 导出当前电脑最新 Codex / Claude Code 会话
- 写入私有数据仓库
- 自动 commit + push 私有数据仓库

这样另一台电脑下次点 Start 就能继续接上。

## 数据目录长什么样

私有数据仓库里会有：

```text
ai-session-data\
  data\
    sessions\
    archived_sessions\
    claude\
    session_summaries\
    session_index.jsonl
```

含义：

- `sessions`：Codex 当前历史
- `archived_sessions`：Codex 归档历史
- `claude`：Claude Code 历史
- `session_summaries`：Kimi 总结缓存
- `session_index.jsonl`：Codex 索引文件

## 以后回家电脑怎么用

如果家里电脑已经有这两个仓库：

1. 打开 `ai-session-explorer`
2. 确认 `.env.local` 里 `SESSION_DATA_ROOT` 指向家里电脑上的 `ai-session-data\data`
3. 双击 `Start-AISessionWork.bat`
4. 双击 `Open-SessionExplorer.bat`

如果家里电脑还没有新仓库：

1. clone 公开工具仓库 `ai-session-explorer`
2. clone 私有数据仓库 `ai-session-data`
3. 创建 `.env.local`
4. 点 `Start-AISessionWork.bat`

## 可选：claude-replay

只有使用“打开 Replay Editor”时才需要：

```powershell
npm install -g claude-replay
```

不安装不影响检索台的主要功能。

## 出错时先看这三件事

### 1. 是否配置了 `SESSION_DATA_ROOT`

必须有：

```env
SESSION_DATA_ROOT=D:\00容器\ai_sys\ai-session-data\data
```

不能写成工具仓库自己的 `data`。

### 2. 两个仓库是否都能 pull

```powershell
git -C D:\00容器\ai_sys\ai-session-explorer pull
git -C D:\00容器\ai_sys\ai-session-data pull
```

### 3. GitHub 是否已登录

如果用 GitHub CLI：

```powershell
gh auth status
```

如果没有登录，执行：

```powershell
gh auth login
```
